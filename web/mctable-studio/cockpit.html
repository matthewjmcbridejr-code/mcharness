Safety Panel

Prototype guardrails and backend truth
No public worker launch
disabled
There is no public launch UI and no arbitrary worker execution path in this cockpit.
Fake-worker-only
allowed
The command dropdown only exposes fake-worker-success, fake-worker-fail, and fake-worker-sleep.
No shell=True
verified
The worker runner uses the existing allowlisted subprocess path and does not expose a shell command box.
MCP local-only
verified
The MCP layer is local and stdio-scoped, not a remote public interface.
Real agents disabled
blocked
AGY, Grok, Codex, and Claude are not launched from this cockpit.
Honest backend gaps
visible
Live API status can show LangGraph and SQLite checkpointing as unavailable when the backend really reports that state. The cockpit mirrors the payload instead of hiding it.
Captain Mode
ready
Captain Mode stores prompt queues, minion tasks, evidence records, hard gates, and next actions through /api/marius/captain/runs. Planned acceptance commands are text only, and there is no real agent launch button.
This prototype is deliberately thin: it renders the live backend state, persists nothing itself, and avoids any hidden mock data.
