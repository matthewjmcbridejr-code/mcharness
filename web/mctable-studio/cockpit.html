How it works

One clear loop from planning to proof. The point is not to be clever. The point is to make agent work auditable and safe to supervise.

6-step control flow
1

Plan

Capture the objective and define the task in a way that can be reviewed before anything moves.

2

Queue

Break the work into prompt queue items so the operator can see the shape of the run.

3

Assign

Route the right bounded minion to the right scope and keep the hands-on control surface visible.

4

Collect Evidence

Capture text evidence and operator notes before the workflow is allowed to continue.

5

Gate

Use proof gates to block the wrong next step and make the reason for the block explicit.

6

Continue / Block

Continue only when the safety model is satisfied; otherwise stop and keep the failure visible.

Safety model

The public demo is designed to be credible without being dangerous. It shows control boundaries clearly and does not pretend to run autonomous production agents.

Operator-first, safety-first

What stays true

✓Public demo does not launch agents.
✓Prompt exports are text-only and easy to review.
✓command_request is blocked when it crosses the safety line.
✓fake-worker is the only executable path in the RC.
✓No unsupervised shell execution is available.
